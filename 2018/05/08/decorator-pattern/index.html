Decorator Pattern

Posted on 2018-05-08 | In Design Pattern
Introduction to decorator pattern with examples.

Concept

Decorator pattern allows user to add new functionality to an existing object without altering its structure. For example, add caching function to the object which requires a cache to improve the efficiency.

Example

  1. Define an interface: Computable<A, V>. The interface has an input of type A and result of type V.
  1. Every computer which wants to have a caching function needs to implement the Computable interface.
  1. Now the ExpensiveComputer doesn’t have the caching function, but we can add a decorator to make it happen.

Explanation:

  1. The thread first checks the cache if there is a existing future.
  2. If yes, then call future.get() to return the result.
  3. If the computation is complete, the get() function will return the result immediately, otherwise it will block until the result has been calculated.
  4. If there is no existing future task, create one and run it.
  • Why while(true)? => The future.get() will throw CancellationException because of the interruption of the thread.
