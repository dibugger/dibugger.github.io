Java类基础

Posted on 2017-12-27 | In Java
Java类基础

基础

类的修饰符有两种:

  • public: 访问无限制
  • default: 同一个包内可见

但变量、方法或构造函数的修饰符还多两种：

  • protected: 同一个包内或不同包内的子类可见
  • private: 同一类内可以访问

内部类

为什么要使用内部类

内部类可以实现多重继承关系，因为每个内部类都可以独立地继承一个类，而无论外部类是否已经继承了某个类。

  1. 内部类可以用多个实例，每个实例都有自己的状态信息，并且与其他外围对象的信息相互独立。
  2. 在单个外围类中，可以让多个内部类以不同的方式实现同一个接口，或者继承同一个类。
  3. 创建内部类对象的时刻并不依赖于外围类对象的创建。
  4. 内部类并没有令人迷惑的“is-a”关系，他就是一个独立的实体。
  5. 内部类提供了更好的封装，除了该外围类，其他类都不能访问。

内部类基础

内部类可以直接访问外部类的任意变量或方法而不管该变量或方法是否由private修饰。这是因为在实例化内部类时，内部类对象会捕获一个外部类对象的引用。直接使用OuterClassName.this可以产生一个正确引用外部类的引用。该引用是JVM为内部类的构造器添加一个参数，参数的类型为指向外部类型的一个引用。

当成员内部类拥有和外部类同名的成员变量或者方法时，会发生隐藏现象，即默认情况下访问的是成员内部类的成员。如果要访问外部类的同名成员，需要以下面的形式进行访问：

  1. 外部类.this.成员变量
  2. 外部类.this.成员方法

内部类的修饰符可以是public, protected, private和default，这与顶级类不同，原因是内部类实际上可以看做外部类的一个成员。

内部类分类

成员内部类

最普通的内部类，是外围类的成员，可以无限制访问外围类的所有成员属性和方法。外围类访问内部类时必须通过内部类的实例来访问。

  1. 不能存在任何的static变量和方法
  2. 只有创建外部类的实例才能创建内部类

局部内部类

有这样一种内部类，它是嵌套在方法和作用于内的，对于这个类的使用主要是应用与解决比较复杂的问题，想创建一个类来辅助我们的解决方案，到那时又不希望这个类是公共可用的，所以就产生了局部内部类，局部内部类和成员内部类一样被编译，只是它的作用域发生了改变，它只能在该方法和属性中被使用，出了该方法和属性就会失效。局部内部类就像是方法里面的一个局部变量一样，是不能有public、protected、private以及static修饰符的。

匿名内部类

类似于new Comparator {
xxx
}

静态内部类（嵌套内部类）

非静态内部类在编译完成之后会隐含地保存着一个引用，该引用是指向创建它的外围内，但是静态内部类却没有。没有这个引用就意味着：

  1. 它的创建是不需要依赖于外围类的。
  2. 它不能使用任何外围类的非static成员变量和方法。
  3. 静态内部类中可以存在static的变量和方法

抽象类

  1. 抽象类提供了继承的概念，不能被实例化，必须由其子类完成实例化的工作。
  2. 抽象方法必须由子类进行重写。
  3. 子类的抽象方法不能与父类的抽象方法同名。
  4. abstract和final不能并列修饰一个类。

接口

接口的修饰符只有public和default两种。且接口中的变量只能为public static final变量名：

  • final: 接口是最高层次的抽象，相当于一个协议，协议是每一个实现接口的类必须遵守的，如果变量不设为final，则每个实现该接口的类都可以改变这个变量的值，违反了OCP的原则。
  • static: 由于Java不支持多继承，但支持实现多个接口。如果C实现了接口A和B，且A和B中都有变量V，如果V不是static的，那么无法知道V是来自于A或者是B。
  1. 不能实例化一个接口。
  2. 所有方法都是public的。
  3. 所有变量都是static final的。
  4. 不存在实现的方法（Java8提供的default接口）
  5. 实现接口的非抽象类必须实现该接口所有方法。抽象类则不需要。
  6. 可以声明一个接口变量，如使用instanceOf来判断一个对象是否实现了某个特定的接口。

与抽象类的区别

  1. 抽象类是对类抽象，接口对行为抽象。
  2. 抽象类是一种继承关系，父类和子类之间必须存在“is-a”关系。接口是一种协议，或者是行为的规范，所有实现接口的类都必须遵守。
