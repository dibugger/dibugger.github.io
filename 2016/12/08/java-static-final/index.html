Java中static和final的作用和用法

Posted on 2016-12-08 | In Java
Static和final的用法总结

Static

static是静态修饰符，所谓静态修饰符，就是指程序编译后所分配的内存一直存在，直到程序退出后才释放该内存。在Java中没有全局变量的概念。使用static修饰的类成员可以直接通过(.)来访问，而不需要新建对象。被static修饰的成员变量和方法独立于该类的任何对象，也就是说静态变量不依赖于任何对象而存在，被类的所有实例所共享。在JVM加载类的时候，Java虚拟机就可以在运行时数据区内找到被static修饰的变量和方法，而不需要实例化任何对象。此外，用static修饰的静态代码块，在Java虚拟机加载类时就会执行。

静态变量

静态变量在内存中只有一个拷贝，JVM只为静态变量分配一次内存，在加载类的过程中完成该静态变量的内存分配，可以直接用类名访问，但也可以通过实例访问。

而实例变量是在创建对象时才会分配内存，实例对象可以在内存中有多个拷贝。各个对象的实例变量之间相互不影响。

静态方法

静态方法可以直接通过类名调用，被所有实例共享，在静态方法中不能使用this和super关键字（因为this和super对应了当前对象），也不能访问所属类的实例对象和实例方法，（也就是static只能调用static）。而且因为static方法独立于任何实例，因此static方法必须被实现，而不能是抽象的abstract。

静态代码块

在Java虚拟机加载类时就会被执行，且只能被执行一次，如果有多个静态代码块，则按顺序执行。

一个static的例子

定义一个Person类，其中有静态变量planet，实例变量age，静态方法getPlanet()，实例方法getAge()。

  • 访问静态变量planet-> String humanPlanet = Person.planet;
  • 访问静态方法getPlanet-> String humanPlanet = Person.getPlanet();
  • 访问实例变量age-> Person diluo = new Person(); int diluoAge = diluo.age();
  • 访问实例方法getAge()-> int diluoAge = diluo.getAge();

静态类（静态内部类）

当我们要定义一个静态类的时候，只有一种情况，那就是静态内部类。在类的外部定义一个Static的类，编译器会报错。

  1. 静态内部类只能访问静态的成员变量和方法，而不能访问普通成员和方法。普通内部类可以访问任何外部类的成员变量和方法（包括静态和非静态）。
  2. 静态内部类中可以声明普通成员变量和方法，而普通内部类中不能声明静态成员变量和方法。
  3. 静态内部类可以单独初始化。

但普通内部类必须先实例化外部类。（这是因为静态类可以直接由类名访问，与静态变量、方法等是一致的）

使用场景：当外部类需要知道内部类，而内部类无需外部类的资源，且内部类可以单独创建时。

《Effective Java》中有详细描写。

final

final关键词意思为“终结的、不可修改的”，被final定义的任何东西都不能被覆盖和修改。

final类

不能被继承，成员方法也默认为final的。如果设计时一个类的设计细节不能被改变，也不能有子类，不能被拓展，那么应该设计成一个final类。

final方法

如果一个类不允许子类覆盖其某个方法，那么应该将该方法声明为final方法。

  1. final方法不允许被改变，其意义和实现一直被保留
  2. 编译器遇到final方法时，不会为其启用多态，因而执行效率非常高
  3. final方法可以被继承，不能被重写(override)

final变量

final变量是个常量。一旦被初始化，就一定不能改变。但可以先声明，在类初始化的同时初始化这个final变量。final修饰的如果是基本数据类型(Primitive Type)，则这个变量不能改变；修饰的如果是引用类型(Object Reference Type)，在引用类型中存放的是地址，所以final只是使其地址不可改变，这个地址所指的对象、数组都是可以改变的。

final参数

传递一个final参数，这个参数不允许被改变。在多线程的环境下，可以保证传入的参数不会被另一线程改变从而改变方法的行为。

static和final一起用

static final代表了全局变量，该变量不可改变，可以通过类名访问。

当它们修饰的是容器对象时，如HashMap，虽然实例不可改变，但容器对象中存放的对象则是可以改变的。
