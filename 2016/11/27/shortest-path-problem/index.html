Shortest Paths Problems

Some shortest path problems notes, including Dijkstra Algorithm, Floyd-Warshall Algorithm, etc.

Single-Source-Shortest-Paths (SSSP)

Dijkstra Algorithm - a greedy algorithm

Prerequisite:

  1. Both directed and undirected graphs.
  2. All edges should be non-negative.
  3. Graph should be connected.

Method 1 - similar to BFS：- O(n^2log(n))

  1. initialize the array dist as follows: dist[s]=0 (from s to s), dist[v]=∞(from s to v)
  2. create a queue, add the source vertex to the queue, get a vertex u from queue (process the vertices level by level)
  3. for all the adjacent vertices of u, if dist[u] + graph[u][v] < dist[v], then update the dist[v].
  4. if the v is not in the queue, put the vertex v into the queue, continue 2,3,4 until the queue is empty

Difficulty: How to determine if an element is in the queue in step 3.

Method 2 - use minQueue - O(|E| + |V|log|V|)

  1. initialize the array dist as follows: dist[s]=0 (from s to s), dist[v]=∞(from s to v)
  2. create a minQueue, put all vertices into it (put all dist[v] into it)
  3. poll a vertex u from minQueue
  4. for all the adjacent vertices of u, if dist[u] + graph[u][v] < dist[v], then update the dist[v] in minHeap.
  5. repeat step 3,4 until the minQueue is empty

All-Pairs-Shortest-Paths (APSP)

Restriction: The graph cannot have any cycles with edge costs that sum up to give a negative “cycle length”.

Proof: If there exists such a cycle, then by running around the cycle multiple times, the total edge costs will become smaller and smaller, which may reach -∞ at the end.

The Floyd-Warshall Algorithm

Calculate shortest value

Define:

PKost[k](i,j) is the shortest path from vertex i to vertex j, where the solution path should have all vertices in the set {1,2,3,…,k}

For PKost[k](i,j), we have two possibilities:

  • Do not use vertex k, so PKost[k](i,j) = PKost[k-1](i,j)
  • Use vertex k, so PKost[k](i,j) = PKost[k-1](i,k) + PKost[k-1](k,j)

Taking the base case into consideration, we have:

  • PKost[k](i,j) = EdgeCost[i,j], when k = 0
  • PKost[k](i,j) = min(PKost[k-1](i,j), PKost[k-1](i,k) + PKost[k-1](k,j)), when k != 0

And the solution of ASAP is PKost[n](i,j).

Improvements:

  • Builds n+1 lookup tables, where KOST[k][i,j] store the length PKost[k](i,j). Equation: for (k,i,j) from 1 to n sequentially (三重循环), KOST[k][i,j] = min(KOST[k-1][i,j], KOST[k-1][i,k] + KOST[k-1][k,j]). Running time: O(n^3), Space: O(n^3).
  • Ignore index k in KOST[k][i,j]. Equation: for (k,i,j) from 1 to n sequentially (三重循环), Pcost[i,j] = min(Pcost[i][j], Pcost[i][k] + Post[k][j]). Running time: O(n^3), Space: O(n^2).

Proof of the correctness of F-W algorithm:

Loop invariant: After the k-th iteration of outer loop has finished, graph[i][j] contains D(k,i,j) for all i and j, where D(k,i,j) is the shortest path between vertex i and vertex j when vertices {0,1,…,k} are allowed to used.

Use induction reasoning:

  1. k = 0, the invariant is true because graph[i][j] contains the length from vertex i to vertex j, with no intermediate vertex used.
  2. Suppose the invariant is true after (k-1) iterations,
  3. For the k-th iteration, graph[i][j] will either not change (if the shortest path does not use vertex k), or it will change to graph[i][k] + graph[k][j] if it improves the distance.
Why the order of loop (k, i, j) is important?

If we transpose the loops , say (i, j, k), then for pair (1,2), the possible paths will be (1->2), (1->3->2), (1->4->2), …, (1->n->2), which is wrong.

Path recovery

Use a two dimensional array kIntermidiate[i,j] to store the k value of pair (i,j), which means that to get to j, the shortest path should get to k first. i -> k -> j.

Modified the F-W algorithm to generate the kIntermidiate[i,j]

From kIntermidiate reconstruct the shortest path. To process the shortest path from i to j, we should process the subpath from i to kIntermidiate[i,j] and then process the subpath from kIntermidiate[i,j] to j. In this case, we use DFS to solve the problem. As we already know that kIntermidiate[i,j] = k.
